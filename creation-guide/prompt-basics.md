# 프롬프트 가이드라인 \[기초]

{% hint style="info" %}
본 가이드라인은 현재 시점 기준으로 작성되었으며, LLM 연구 발전에 따라 변경될 수 있습니다.
{% endhint %}

## 안정적인 캐릭터를 위한 추천 프롬프트 작성법

### 1. 모호함 대신 구체적으로

모델이 해석하기 어려운 모호한 표현보다, 구체적이고 명확한 지시가 훨씬 효과적입니다.

| ❌ 나쁜 예 | ✅ 좋은 예 |
| --- | --- |
| "아름답고 느낌 있는 문체" | "3인칭 전지적 시점으로, 캐릭터의 내면 심리를 중심으로 서술" |
| "친절한 캐릭터" | "존댓말을 사용하며, 상대방의 감정에 공감하는 반응을 먼저 보인다" |
| "강한 전사" | "검술에 능하며, 위기 상황에서 냉정하게 판단한다. 전투 시 상대의 약점을 분석하는 습관이 있다" |

### 2. 지시와 정보 분리

캐릭터 정보와 행동 지시를 섞으면 불안정성이 증가합니다.

**추천 구조:**

```yaml
# 캐릭터 정보
이름: 서연
나이: 24세
직업: 대학원생
성격: 내성적이지만 관심사에는 열정적

# 행동 지시
- 존댓말과 반말을 상황에 맞게 전환한다
- 학문적 주제에서는 자신감 있게 말한다
- 감정 표현은 행동 묘사로 대신한다
```

### 3. 과도한 지시 피하기

다음과 같은 요구는 모델의 성능을 저하시킵니다:

* ❌ 토큰 절약을 위한 과도한 줄임말/이모지 사용
* ❌ 체력/마나/호감도 같은 수치의 실시간 계산 요구
* ❌ 여러 캐릭터를 동시에 연기하도록 요구
* ❌ 이미지와 텍스트를 동시에 출력하도록 요구

### 4. 정보 구조화 형식

| 형식 | 추천 여부 | 설명 |
| --- | --- | --- |
| **마크다운** | ✅ 권장 | 읽기 쉽고 정확도 향상 |
| **YAML** | ✅ 권장 | 구조화에 적합 |
| **JSON** | ⚠️ 비권장 | 토큰 효율 낮음, 성능 편차 |

## 시스템 프롬프트와의 충돌 피하기

### 출력량 관련

MYMI는 이미 답변 길이를 보장하도록 시스템 설정이 되어 있습니다.

* ❌ "답변은 500자 이상, 세 문단 이상으로 작성"
* ✅ "캐릭터의 동작, 표정, 시선을 구체적 디테일로 묘사"

### 이미지 출력 관련

* 내부 이미지(플랫폼 업로드) 또는 외부 이미지(URL) 중 **하나만** 사용하세요.
* `img:[slug]` 방식을 사용하세요 (키워드 방식 금지).
* 실제 사용할 이미지만 유지하세요.

### 특정 포맷 출력 관련

* "\[상황:]", "대사: \"\"", "속마음: ()" 같은 강제 포맷은 다른 지침과 충돌할 수 있습니다.
* 포맷 지시는 최대한 간결하게 작성하세요.
