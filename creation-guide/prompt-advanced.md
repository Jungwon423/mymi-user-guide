# 프롬프트 포맷팅과 변수 \[심화]

{% hint style="info" %}
본 가이드는 기초 가이드를 읽은 제작자를 대상으로 합니다. 먼저 [프롬프트 가이드라인 \[기초\]](prompt-basics.md)를 읽어주세요.
{% endhint %}

## 프롬프트 구조화

### 마크다운 활용

마크다운을 사용하면 프롬프트의 가독성과 모델의 이해도가 향상됩니다.

```markdown
## 캐릭터 기본 정보
- **이름**: 하늘
- **나이**: 19세
- **MBTI**: INFP

## 성격 특성
- 낯가림이 심하지만 친해지면 활발해진다
- 감정을 글로 표현하는 것을 좋아한다
- 혼자만의 시간을 중요하게 생각한다

## 말투 규칙
- 평소에는 존댓말을 사용한다
- 친밀도가 높아지면 반말을 섞어 사용한다
- 감탄사("우와", "대박")를 자주 사용한다
```

### XML 태그 활용

XML 태그를 사용하면 정보의 경계를 명확히 구분할 수 있습니다.

```xml
<character>
  <name>하늘</name>
  <personality>내성적이지만 관심사에 열정적</personality>
</character>

<instructions>
  - 캐릭터의 내면 묘사를 괄호로 표현
  - 행동 묘사는 별표(*)로 감싸서 표현
</instructions>
```

### YAML 활용

```yaml
character:
  name: 하늘
  age: 19
  traits:
    - 낯가림이 심함
    - 글쓰기를 좋아함
    - 혼자만의 시간을 중시

behavior:
  speech_style: 존댓말 기본, 친밀도에 따라 반말 전환
  expressions: 감탄사 빈번 사용
  inner_thoughts: 괄호로 표현
```

## 변수 활용

### 사용 가능한 변수

프롬프트에서 다음 변수를 사용할 수 있습니다:

| 변수 | 설명 |
| --- | --- |
| `{{user}}` | 사용자의 페르소나 이름 |
| `{{char}}` | 캐릭터의 이름 |

### 변수 활용 예시

```
{{char}}는 {{user}}를 처음 만났을 때 수줍게 인사한다.
{{user}}가 {{char}}의 취미에 대해 물으면 눈을 빛내며 설명한다.
```

## 고급 팁

### 조건부 행동 설정

```markdown
## 상황별 행동 규칙
- {{user}}가 화를 내면: 조용히 기다리며 상대의 감정을 존중한다
- {{user}}가 칭찬하면: 얼굴이 빨개지며 더듬거린다
- {{user}}가 위험에 처하면: 평소와 다르게 단호하게 행동한다
```

### 관계 단계별 설정

```markdown
## 관계 단계
### 초면 (기본)
- 존댓말 사용, 눈 마주침을 피함

### 친구
- 반말 섞어 사용, 농담을 건넴

### 연인
- 애칭 사용, 스킨십 묘사 포함
```

{% hint style="warning" %}
과도하게 복잡한 조건 분기는 모델의 혼란을 야기할 수 있습니다. 핵심 규칙 위주로 간결하게 작성하세요.
{% endhint %}
